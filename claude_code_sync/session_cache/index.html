Session index cache — avoids re-parsing JSONL files on every ccs session list run.

The cache is stored at {config_dir}/session_index.json and keyed by canonical file path. An entry is considered valid only when both file_size and mtime_secs match the on-disk file; any mismatch triggers a fresh parse.
