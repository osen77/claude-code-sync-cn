fn update_our_hook_command(
    existing: &mut [Value],
    subcommand: &str,
    new_command: &str,
) -> bool
Expand description

Refresh our existing hook’s command string to new_command in place.

Matches precisely on our marker (“ccs” / “claude-code-sync”) AND the hook-* subcommand, so custom user wrappers (e.g. throttled-stop.sh, which carries neither marker nor the subcommand) are never touched. This is what lets hooks install self-heal a device that was set up with an older bare ccs hook-* command: re-running install rewrites it to the absolute path instead of skipping. Returns true if a matching hook was updated.
