Remove a session’s local file and sync-repo copy, returning a tombstone record for the caller to batch-persist.

Unlike delete_session_with_commit, this does NOT save the tombstone registry or commit — the caller is expected to accumulate records and perform a single save + commit at the end (used by batch cleanup so the history gets one commit instead of N).

Returns None when the local file could not be removed (the session is left untouched and the caller can continue with the rest).
